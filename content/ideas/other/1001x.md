# Setup iceman (hermes-agent) on a new VM ‚Äî clean clone of 1001x

## Context

Standing up a new agent called **iceman** on a fresh VM, running the hermes-agent (Python) platform ‚Äî the same engine as the existing 1001x install.  
 This is a clean start ‚Äî no session history, no workspace state, no agent memory. The personalities and PR-review plugin code live in the  
 `ask-myfi/oc-personalities` GitHub repo, cloned into the runtime dir and refreshed by the `sync-oc-personalities` cron. The two hermes-agent
customizations that aren't upstream (`gateway/platforms/slack.py` SlackApiError demotion, `hooks/rtk-rewrite/handler.py` None-guard) need to be  
 re-applied unless the source fork already carries them. A new Slack app/bot account is being created for `@iceman`.

## Stepwise procedure

### Step 1 ‚Äî VM prereqs

- Fresh Ubuntu/Debian VM, ‚â•30 GB free disk (hermes is heavier than openclaw ‚Äî sessions + workspace can grow fast)
- Install: `python3.11+` with `venv`, `pip`, `git`, `nodejs` (v22.x ‚Äî needed for the Node-based MCPs), `npm` (10.x), `systemd`, `cron`,  
  `openssh-client`, `rsync`
- Outbound network: Anthropic, Together (MiniMax + Kimi fallbacks), Slack, GitHub (SSH:22 + HTTPS:443), Linear, Codemagic, GCP
- Time sync (`chrony` / `systemd-timesyncd`)  


### Step 2 ‚Äî Create system user and group

```bash
sudo groupadd icemangrp
sudo useradd --gid icemangrp --home-dir /home/iceman --create-home --shell /bin/bash iceman
sudo usermod -aG icemangrp kiran
sudo usermod -aG icemangrp saravanan
sudo usermod -aG icemangrp gokul
```

### Step 3 ‚Äî Clone the hermes-agent source

```bash
# Get the fork URL from the current VM's .git/config (it's a fork of nousresearch/hermes-agent)
sudo -u iceman git clone <ask-myfi-hermes-fork-URL> /home/iceman
# That places source directly under /home/iceman/ (matching the 1001x convention)
```

If the fork doesn't already have the two recent patches (slack `is_archived` demotion + rtk-rewrite None-guard), grab them from the current VM's  
 `gateway/platforms/slack.py.bak.20260514-*` and `~/.hermes/hooks/rtk-rewrite/handler.py.bak.20260514-*` (the _non_-bak files are the patched  
 versions).

### Step 4 ‚Äî Python venv + install

```bash
sudo -u iceman python3 -m venv /home/iceman/.venv
sudo -u iceman /home/iceman/.venv/bin/pip install --upgrade pip
# Try the bulk install first; if it hits ResolutionImpossible, fall back to per-extra (the upstream
# install script does this automatically ‚Äî check the repo for an install.sh or similar)
sudo -u iceman /home/iceman/.venv/bin/pip install -e '.[all]'
# Verify
sudo -u iceman /home/iceman/.venv/bin/hermes --version
```

### Step 5 ‚Äî Initialise a clean runtime state dir

```bash
sudo -u iceman mkdir -p /home/iceman/.hermes/{cron/output,hooks,scripts,backups,plugins,repos,mcp-servers,workspace/repos}
sudo chmod 700 /home/iceman/.hermes
sudo chown -R iceman:icemangrp /home/iceman/.hermes
```

### Step 6 ‚Äî Bring across the personalities/PR-review content (only thing migrated)

All the `.md` files (SOUL.md, IDENTITY.md, MEMORY.md, AGENTS.md, USER.md, TOOLS.md, and ~30 `*_PERSONALITY.md` files) live in the  
 `ask-myfi/oc-personalities` repo. The hermes cron `sync-oc-personalities` keeps it refreshed.

**6a. Generate a deploy key for iceman and add it to the GitHub repo**

```bash
sudo -u iceman ssh-keygen -t ed25519 -f /home/iceman/.ssh/id_ed25519 -N "" -C "iceman@<new-vm>"
sudo -u iceman cat /home/iceman/.ssh/id_ed25519.pub
```

Add the printed pubkey to `ask-myfi/oc-personalities` ‚Üí Settings ‚Üí Deploy keys (read-only). Same key works for the hermes-agent fork if you cloned via
SSH in Step 3.

```bash
sudo -u iceman ssh -o StrictHostKeyChecking=accept-new -T git@github.com
```

**6b. Clone the personalities repo**

```bash
sudo -u iceman git clone git@github.com:ask-myfi/oc-personalities.git \
  /home/iceman/.hermes/repos/oc-personalities
```

**6c. Carry over the sync script** (hermes uses a slightly different sync script than openclaw ‚Äî copy it, patch paths)

```bash
sudo scp user@old-vm:/home/hermes-agent/.hermes/scripts/sync-oc-personalities.sh \
  /home/iceman/.hermes/scripts/sync-oc-personalities.sh
sudo chown iceman:icemangrp /home/iceman/.hermes/scripts/sync-oc-personalities.sh
sudo chmod 700 /home/iceman/.hermes/scripts/sync-oc-personalities.sh
sudo sed -i 's|/home/hermes-agent/|/home/iceman/|g' /home/iceman/.hermes/scripts/sync-oc-personalities.sh
sudo sed -i 's|systemctl restart hermes-gateway|sudo systemctl restart iceman-gateway|' \
  /home/iceman/.hermes/scripts/sync-oc-personalities.sh
```

**6d. Allow iceman to restart its own gateway without a password**

```bash
echo 'iceman ALL=(root) NOPASSWD: /bin/systemctl restart iceman-gateway' \
  | sudo tee /etc/sudoers.d/iceman-restart
sudo chmod 440 /etc/sudoers.d/iceman-restart
```

### Step 8 ‚Äî Secrets: `/home/iceman/.hermes/.env`

Mode `600`, owned `iceman:icemangrp`. Pull from secret store:

- `ANTHROPIC_API_KEY` ‚Äî **required** (primary + Opus fallback)
- `TOGETHER_API_KEY` ‚Äî **strongly recommended** (MiniMax + Kimi fallbacks; only real Anthropic-billing-outage redundancy)
- `OPENAI_API_KEY` ‚Äî optional; present on current 1001x but not in main provider chain (some auxiliary use)
- `SLACK_BOT_TOKEN` ‚Äî new app's bot token (Step 10)
- `SLACK_APP_TOKEN` ‚Äî new app's Socket Mode token
- `LINEAR_API_KEY` ‚Äî Linear MCP
- `GH_TOKEN` ‚Äî GitHub MCP, must have repo access
- `CODEMAGIC_API_KEY` ‚Äî Codemagic MCP

### Step 14 ‚Äî systemd unit

Create `/etc/systemd/system/iceman-gateway.service` based on the current `hermes-gateway.service`:

```ini
[Unit]
Description=iceman Hermes AI Agent Gateway
After=network-online.target
Wants=network-online.target

[Service]
User=iceman
Group=icemangrp
WorkingDirectory=/home/iceman
ExecStart=/home/iceman/.hermes/scripts/start-gateway.sh
Restart=always
RestartSec=5
TimeoutStopSec=30
TimeoutStartSec=60
KillMode=control-group
Environment=HOME=/home/iceman
Environment=HERMES_HOME=/home/iceman/.hermes

[Install]
WantedBy=multi-user.target
```

The `start-gateway.sh` wrapper is what loads secrets (GCP + .env fallback) ‚Äî copy it across:

```bash
sudo scp user@old-vm:/home/hermes-agent/.hermes/scripts/start-gateway.sh \
  /home/iceman/.hermes/scripts/start-gateway.sh
sudo chown iceman:icemangrp /home/iceman/.hermes/scripts/start-gateway.sh
sudo chmod 700 /home/iceman/.hermes/scripts/start-gateway.sh
sudo sed -i 's|/home/hermes-agent/|/home/iceman/|g' /home/iceman/.hermes/scripts/start-gateway.sh
```

Then:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now iceman-gateway
```
